# 环境配置指南

## 1. JDK 17 安装配置

### 下载JDK 17

1. 访问 Oracle 官方下载页面：https://www.oracle.com/java/technologies/downloads/#jdk17-windows
2. 选择 "Windows x64 Installer"
3. 点击下载，需要登录Oracle账号（可免费注册）

### 安装JDK

1. 双击下载的 `jdk-17_windows-x64_bin.exe` 文件
2. 按照安装向导进行安装
3. 建议使用默认安装路径：`C:\Program Files\Java\jdk-17`

### 配置环境变量

1. 右键点击 "此电脑" → "属性"
2. 点击 "高级系统设置" → "环境变量"
3. 在 "系统变量" 区域点击 "新建"
   - 变量名：`JAVA_HOME`
   - 变量值：`C:\Program Files\Java\jdk-17`（根据实际安装路径调整）
4. 找到并编辑 "Path" 变量
   - 在变量值开头添加：`%JAVA_HOME%\bin;`
5. 点击 "确定" 保存所有设置

### 验证安装

1. 打开命令提示符（Win+R → 输入 "cmd" → 回车）
2. 输入命令：`java -version`
3. 若显示如下信息，则安装成功：
   ```
   java version "17.0.x"
   Java(TM) SE Runtime Environment (build 17.0.x+xx)
   Java HotSpot(TM) 64-Bit Server VM (build 17.0.x+xx, mixed mode, sharing)
   ```

## 2. MySQL 安装配置

### 下载MySQL

1. 访问 MySQL 官方下载页面：https://dev.mysql.com/downloads/installer/
2. 下载 "MySQL Installer for Windows"

### 安装MySQL

1. 双击安装包，选择 "Developer Default" 安装类型
2. 按照向导完成安装
3. 设置root用户密码为：`root`（必须使用此密码，否则需要修改项目配置）
4. 确保MySQL服务启动

### 创建数据库

1. 打开 MySQL Command Line Client
2. 输入密码：`root`
3. 执行命令：
   ```sql
   CREATE DATABASE employment_system CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   ```
4. 验证数据库创建成功：
   ```sql
   SHOW DATABASES;
   ```

## 3. Maven 安装配置（可选）

### 下载Maven

1. 访问 Maven 官方下载页面：https://maven.apache.org/download.cgi
2. 下载 "Binary zip archive" 版本

### 安装Maven

1. 解压下载的 zip 文件到 `C:\Program Files\Apache\Maven` 目录
2. 配置环境变量：
   - 新建系统变量：`MAVEN_HOME`，值为 `C:\Program Files\Apache\Maven\apache-maven-3.x.x`
   - 在 Path 变量中添加：`%MAVEN_HOME%\bin`

### 验证安装

1. 打开命令提示符
2. 输入命令：`mvn -version`
3. 若显示Maven版本信息，则安装成功

## 6. 常见问题解决

### 1. Java环境变量配置错误

- 症状：`java` 命令未找到
- 解决：重新检查环境变量配置，确保路径正确

### 2. MySQL连接失败

- 症状：启动时出现数据库连接错误
- 解决：检查MySQL服务是否启动，密码是否正确

### 3. 端口被占用

- 症状：启动时出现端口8080被占用
- 解决：关闭占用端口的程序，或修改 `application.properties` 中的端口配置

### 4. 构建失败

- 症状：`mvn clean package` 执行失败
- 解决：检查网络连接，确保Maven能下载依赖

## 7. 常见问题解决

### 1. Java环境变量配置错误

- 症状：`java` 命令未找到
- 解决：重新检查环境变量配置，确保路径正确

### 2. MySQL连接失败

- 症状：启动时出现数据库连接错误
- 解决：检查MySQL服务是否启动，密码是否正确（应为123456）

### 3. 端口被占用

- 症状：启动时出现端口8080被占用
- 解决：关闭占用端口的程序，或修改 `application.properties` 中的端口配置

### 4. 构建失败

- 症状：`mvn clean package` 执行失败
- 解决：检查网络连接，确保Maven能下载依赖

### 5. 前端依赖安装失败

- 症状：`npm install` 执行失败
- 解决：检查网络连接，或使用 `npm install --registry=https://registry.npmmirror.com`

### 6. 前端启动失败

- 症状：`npm run dev` 执行失败
- 解决：确保Node.js版本正确，依赖安装成功
